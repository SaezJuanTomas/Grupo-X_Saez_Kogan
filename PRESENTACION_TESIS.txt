1. ¿QUÉ ES GRUPO X?

Grupo X es una plataforma integral de gestión y monitoreo de vulnerabilidades de 
seguridad informática, diseñada como un sistema empresarial que permite a las 
organizaciones identificar, clasificar, asignar y resolver problemas de seguridad 
en su infraestructura de TI.

El proyecto implementa un modelo de flujo de trabajo colaborativo donde diferentes 
roles de usuario pueden trabajar en paralelo: administradores supervisando la 
ejecución general, analistas de seguridad investigando vulnerabilidades específicas, 
y automatización inteligente enriqueciendo datos en tiempo real.


2. HERRAMIENTAS Y TECNOLOGÍAS UTILIZADAS

ARQUITECTURA GENERAL:
  • Modelo cliente-servidor distribuido
  • Base de datos relacional centralizada
  • API REST para comunicación asincrónica
  • Interfaz web responsiva y moderna

FRONTEND (Interfaz de Usuario):
  • React 18+: Framework de JavaScript para interfaces interactivas
  • TypeScript: Tipado estático para mayor seguridad en código
  • Vite: Bundler ultra-rápido (tiempo de compilación <1 segundo)
  • TailwindCSS: Framework de utilidades CSS para diseño moderno
  • Axios: Cliente HTTP para consumir APIs
  • Contexto de Autenticación: Control de sesiones y permisos por rol

BACKEND (Servidor y Lógica):
  • FastAPI: Framework web Python de alto rendimiento (3x más rápido que Django)
  • SQLAlchemy: ORM (Object-Relational Mapping) para gestión de base de datos
  • SQLite: Base de datos relacional embebida, perfecta para MVP
  • JWT (JSON Web Tokens): Autenticación segura sin sesiones de servidor
  • Validación de Esquemas: Pydantic valida datos entrada/salida

AUTOMATIZACIÓN INTELIGENTE (n8n):
  • Integración n8n: Orquestación de workflows sin código
  • Cálculo de IRC: Algoritmo propietario de riesgo de vulnerabilidades
  • Correlación de Activos: Identificación automática de sistemas afectados
  • Webhooks: Comunicación en tiempo real entre sistemas
  • Base de Datos de Amenazas: Integración con CVE y fuentes externas

INFRAESTRUCTURA:
  • Docker: Containerización para despliegue reproducible
  • SQLite: Almacenamiento persistente local
  • APIs RESTful: Estándares modernos de comunicación


3. ARQUITECTURA Y FLUJO DE DATOS

ESTRUCTURA DE LA BASE DE DATOS:

  Empresas (Companies)
  ├── ID, Nombre, Sector, Contacto
  └── Relación 1:N con Vulnerabilidades

  Usuarios (Users)
  ├── ID, Email, Contraseña, Rol (Administrador/Analista)
  ├── Estado Activo/Inactivo
  └── Relación 1:N con Vulnerabilidades asignadas y Comentarios

  Vulnerabilidades (Vulnerabilities)
  ├── ID, CVE (identificador único)
  ├── Descripción técnica detallada
  ├── IRC (Índice de Riesgo Calculado 0-100)
  ├── Severidad (Crítica/Alta/Media/Baja)
  ├── Estado (Detectada/En Análisis/Remediada/Falso Positivo)
  ├── Empresa Afectada
  ├── Analista Asignado
  └── Relación 1:N con Comentarios e Historial

  Comentarios (Comments)
  ├── Contenido, Autor, Timestamp
  └── Vinculados a Vulnerabilidades

  Historial (HistoryLogs)
  ├── Tipo de acción (Creación, Actualización, Asignación)
  ├── Usuario que realizó la acción
  ├── Timestamp exacto
  └── Detalles de cambio


FLUJO DE AUTENTICACIÓN:

  1. Usuario ingresa credenciales (email + contraseña)
  2. Backend verifica credenciales contra base de datos
  3. Si es válido, genera JWT firmado con rol del usuario
  4. Frontend almacena token en localStorage (persistencia)
  5. Cada solicitud posterior incluye token en header Authorization
  6. Backend valida firma del token y permisos según rol
  7. Si token inválido/expirado, usuario es redirigido a login


FLUJO DE GESTIÓN DE VULNERABILIDADES:

  FASE 1 - DETECCIÓN (Automatizada por n8n):
    → n8n consulta fuentes de amenazas externas (CVE, CISA, etc.)
    → Calcula IRC (Índice de Riesgo Completo) usando fórmula propietaria
    → Correlaciona vulnerabilidades con activos de la empresa
    → Crea registros en la base de datos automáticamente
    → Genera alertas para administradores

  FASE 2 - ASIGNACIÓN (Administrador):
    → Admin visualiza vulnerabilidades pendientes en dashboard
    → Distribuye asignaciones a analistas según especialidad
    → Define prioridades y deadlines
    → Registra acciones en historial automáticamente

  FASE 3 - INVESTIGACIÓN (Analista):
    → Analista recibe notificación de asignación
    → Revisa detalles, severidad, IRC, contexto técnico
    → Agrega comentarios con hallazgos y análisis
    → Actualiza estado de la vulnerabilidad
    → Sistema registra cada cambio con timestamp

  FASE 4 - REMEDIACIÓN:
    → Analista actualiza a estado "Remediada" con pruebas
    → Administrador verifica y cierra
    → Historial completo queda registrado para auditoría

  FASE 5 - REPORTERÍA Y ESTADÍSTICAS:
    → Dashboard ejecutivo muestra métricas agregadas
    → Gráficos de tendencias temporales
    → Matriz de severidad vs estado
    → KPIs de tiempo de respuesta

5. AUTOMATIZACIÓN CON n8n

n8n es un motor de automatización sin código que ejecuta workflows complejos. 
En Grupo X se utiliza para:

WORKFLOW PRINCIPAL:
  Nodo 1: Disparador por horario (cada X minutos)
  Nodo 2: Consulta CVSS Base Score
  Nodo 3: Consulta Environmental Factor
  Nodo 4: Consulta Temporal Score
  Nodo 5-10: Enriquecimiento de datos con fuentes externas
  Nodo 11: Cálculo de IRC (Risk Index Calculation)
  Nodo 12: Correlación de activos
  Nodo 13: Envío a API de Grupo X
  Nodo 14: Logging y notificaciones

FÓRMULA IRC (Índice de Riesgo Completo):
  IRC = (CVSS_Base × 0.6) + (Environmental_Factor × 0.3) + 
        (Temporal_Score × 0.1) + (Correlation_Weight × 0.05)

Resultado: puntuación 0-100 que prioriza vulnerabilidades automáticamente

8. ENDPOINTS DE API PRINCIPALES

Autenticación:
  POST   /login                    → Obtener JWT token
  POST   /logout                   → Cerrar sesión

Vulnerabilidades:
  GET    /vulnerabilidades        → Listar todas (filtrado por rol)
  GET    /vulnerabilidades/{id}   → Obtener detalles
  POST   /vulnerabilidades        → Crear nueva (admin)
  PUT    /vulnerabilidades/{id}   → Actualizar (admin/analista)
  DELETE /vulnerabilidades/{id}   → Eliminar (admin)

Usuarios:
  GET    /usuarios                → Listar usuarios (admin)
  POST   /usuarios                → Crear usuario (admin)
  PUT    /usuarios/{id}           → Modificar usuario (admin)

Comentarios:
  POST   /vulnerabilidades/{id}/comentarios  → Agregar comentario
  GET    /vulnerabilidades/{id}/comentarios  → Obtener comentarios

Estadísticas:
  GET    /estadisticas/resumen    → Métricas ejecutivas
  GET    /estadisticas/tendencias → Datos históricos

Historial:
  GET    /historial?filtros       → Registro de auditoría

Las PYMES y empresas medianas enfrentan un problema crítico: la cantidad de 
amenzas de seguridad informática crece exponencialmente, pero sus presupuestos 
no. Mientras que soluciones enterprise como SOAR cuestan $200k+ anuales, la mayoría 
de PYMEs no pueden justificar esa inversión.

Grupo X es la solución que permite a PYMEs y empresas medianas transformar el caos 
de la gestión de vulnerabilidades en un proceso ordenado, automatizado y medible 
**SIN el presupuesto ilimitado** requerido por soluciones enterprise.

Con Grupo X, PYMEs ahora pueden:
  • Acceso asequible: $3k-$30k/año en lugar de $200k+
  • Implementación rápida: <1 mes vs 6 meses de SOAR
  • ROI probado: payback en 6 semanas vs nunca con soluciones caras
  • Escalabilidad: crece con la empresa sin penalización
  • Simplicidad: diseñada para PYMEs, no enterprise bloatware

  Solución Grupo X:
    ✓ Plataforma centralizada que agrupa todas las vulnerabilidades
    ✓ Visibilidad total del panorama de seguridad

    ✓ Trazabilidad completa de cada incidencia

  Situación actual:
    • No hay criterio claro para distribuir análisis entre analistas
    • Algunas personas están sobrecargadas, otras subutilizadas
    • Es difícil saber quién está investigando qué

  Situación actual:
    • Análisis manuales consume 80% del tiempo

    • Clasificación de severidad inconsistente
    • Recálculos repetitivos sin valor agregado

  Solución Grupo X:
    ✓ Integración n8n automatiza detección y enriquecimiento
    ✓ Cálculo de riesgo (IRC) automático y consistente

  Situación actual:
    • Reguladores exigen documentación de investigaciones
